流程图

通过上图我们可以看到，执行最终的判断登录信息和写入权限是由Realm执行的。

  1. 首先调用 Subject.login(token) 进行登录，其会自动委托给 Security Manager，调用之前必须通过 SecurityUtils.setSecurityManager() 设置；
  2. SecurityManager 负责真正的身份验证逻辑；它会委托给 Authenticator 进行身份验证；
  3. Authenticator 才是真正的身份验证者，Shiro API 中核心的身份认证入口点，此处可以自定义插入自己的实现Authenticator 可能会委托给相应的 AuthenticationStrategy 进行多 Realm 身份验证，默认 ModularRealmAuthenticator 会调用 AuthenticationStrategy 进行多 Realm 身份验证；
  4. Authenticator 会把相应的 token 传入 Realm，从 Realm 获取身份验证信息，如果没有返回 / 抛出异常表示身份验证失败了。此处可以配置多个 Realm，将按照相应的顺序及策略进行访问。

Java 虚拟机栈

定义

虚拟机栈描述的是Java方法执行的内存模型，每个方法在执行的时候都会创建一个栈帧用来存放局部变量表、操作数栈、动态链接、方法出口等信息。

局部变量表

局部变量表存放了编译期可知的各种基本数据类型(boolean,byte,char,short,int,long,float,double)对象引用和returnAddress地址类型。
其中局部变量表中long和double会占用2个slot空间。其它的都只占用1个slot空间。

异常创建

由于采用的数据结构是栈。我们知道如果不出栈的话。会一直在栈内压进去，直到找到一个出口。最有名的就是递归容易使栈内存溢出。。。

session管理器

发表于 2017-03-29

提供的功能

  1. 配置方便。对IOC容器是友好的。Shiro的一切（包括会话和会话管理的所有方面）都是基于接口的实现POJOs。
  2. 会话存储。在shiro中都是POJO，所以我们既可以保存到磁盘、缓存，也可以保存到数据库中。
  3. 简单地集群。Shiro的会话可以轻易集群使用任何现成的网络缓存，比如ehcache、redis.
  4. 跨客户端访问。比如用户登录桌面客户端成功之后，我们会话管理器会保存此会话，那么客户从web客户端或者移动客户端登录，也是可以访问到此会话的。
  5. 主机地址保留。在Shiro会话发起时会保存主机的IP地址。
  6. 事件监听。你可以监听会话的生命周期变换的时候的事件。比如用户会话过期时更新用户记录。
  7. SSO
  8. 对 Web 的透明支持。在shiro对web的支持中，继承了HttpSession接口。实现了我们也可以把shiro的会话放入到HttpSession中，并且不用改变代码
  9. 失效/过期支持。

Apache Shiro

发表于 2017-03-29

定义

shiro是一个简单强大并且易于使用的一个权限控制框架，它提供了认证（authentication）, 授权（authorization）,加密（ cryptography）, and session管理1器（session management）。它可被用于任何的应用。比如命令行应用，移动应用，web企业应用

安利一篇好文RBAC新解解释了关于角色的权限控制与关于资源的权限控制的问题

需了解的概念

详细介绍

  1. Principals ：用于验证身份的输入。比如姓名，姓氏，身份证号，用户名

  2. Credentials ： 用于验证身份的输入。密码，生物识别数据，X509证书，

  3. Subject：主体，代表了当前 “用户”，这个用户不一定是一个具体的人，与当前应用交互的任何东西都是 Subject，如网络爬虫，机器人等；即一个抽象概念；所有 Subject 都绑定到 SecurityManager，与 Subject 的所有交互都会委托给 SecurityManager；可以把 Subject 认为是一个门面；SecurityManager 才是实际的执行者；

  4. SecurityManager：安全管理器；即所有与安全有关的操作都会与 SecurityManager 交互；且它管理着所有 Subject；可以看出它是 Shiro 的核心，它负责与后边介绍的其他组件进行交互，如果学习过 SpringMVC，你可以把它看成 DispatcherServlet 前端控制器；

  5. Realm：域，Shiro 从从 Realm 获取安全数据（如用户、角色、权限），就是说 SecurityManager 要验证用户身份，那么它需要从 Realm 获取相应的用户进行比较以确定用户身份是否合法；也需要从 Realm 得到用户相应的角色 / 权限进行验证用户是否能进行操作；可以把 Realm 看成 DataSource，即安全数据源。

功能实现

1. 认证（Authentication）

提供用户身份，我们经常称之为登录，shiro中的使用reaml进行用户登录

认证有如下步骤
(1). 创建令牌
(2). 提交令牌到认证系统
(3). 认证成功，允许访问；认证失败未超过固定次数，返回重新认证；否则直接阻止其访问

授权（Authorization）

对用户进行访问控制，授权粒度支持角色和权限。如果支持到权限，权限的体现可以在菜单，或者按钮或者我们任何已知的资源。如果大家对rest架构有理解的话就很好理解这个权限控制了。

程序计数器

发表于 2017-03-28

程序计数器

1.定义

程序计数器是一块较小的空间，它可以看做是当前线程所执行的字节码的行号指示器。

2.作用

在虚拟机的概念模型里，字节码解释器工作时就是通过改变这个计数器的值来选取下一条需要执行的字节码的指令，分支、循环、跳转、异常处理、线程恢复等基础功能都需要依赖这个程序计数器

3.个人理解

因为处理器的一个核在同一时间只能处理一条线程，所以在多个线程的情况下，运行时的线程会进行线程切换，但是为了在切换完成时恢复到上一次执行的地方，所以每一个线程都需要保存住自己的运行状态。所以每一个线程都有自己的程序计数器，各个计数器互不影响，保存这些计数器等线程私有的区域又称为线程私有区。

tip

  1. 执行java方法的时候，表示的是当前线程所执行的字节码的行号指示器
  2. 执行Native方法的时候，值为0
  3. 线程私有区是唯一一个没有OutOfMemoryError的一个区域
